# LLM 大模型基础

## Transformer的位置编码为什么用正弦余弦函数，而不是直接学习一个位置embedding？
这个问题问的是泛化性。学习到的位置embedding在训练时见过的序列长度范围内表现很好，但如果推理时遇到比训练时更长的序列，超出范围的位置没有对应的embedding，处理不了。
正弦余弦位置编码是确定性的函数，可以外推到任意长度，不依赖训练时见过的长度上限。另外正弦余弦函数的性质让模型相对容易学到位置之间的相对关系——两个位置的编码的内积，只和它们的距离有关，和绝对位置无关。

## RLHF的训练流程，Reward Model怎么训练，PPO的clip机制是什么，为什么要clip？
Reward Model是在人类偏好数据上训练的，输入是一个prompt加上一个response，输出是一个标量分数，表示这个response有多好。训练数据是成对的——同一个prompt下，人类标注者认为哪个response更好，用这个偏好对训练一个排序模型。
PPO的clip是为了限制策略更新的步长。如果每次更新的幅度太大，策略可能跑到一个很差的地方，然后因为分布差异太大，Reward Model给的分数也不再可信。clip把新旧策略的概率比限制在一个范围内，大概是0.8到1.2之间，超出这个范围就截断，防止策略更新太激进。

## Pre-norm和Post-norm的区别，你们用的哪个，为什么？
Post-norm是原始Transformer 用的，LayerNorm放在残差连接之后。Pre-norm是把LayerNorm放在子层之前。
Pre-norm训练更稳定，梯度流更好，现在大多数大模型都用Pre-norm。Post-norm在某些任务上最终效果更好，但训练容易不稳定，需要更仔细的学习率调整。
我们用的Pre-norm，原因就是稳，不容易出幺蛾子，在资源有限的时候不想把时间花在调训练稳定性上。

## 你的模型在线上推理的时候，怎么监控它的输出质量？离线评估指标好，上线之后效果差，你怎么排查？
这个问题问的是分布漂移。离线测试集和线上真实数据的分布不一样，是导致离线好、上线差的最常见原因。排查方向是先看线上的输入数据分布有没有发生变化，再看模型的输出分布，比较和离线的差异，然后针对性地补数据或者做领域适应。

## 如果我是大模型算法的面试官，我会问什么
第一类，问注意力机制为什么要多头，一个头不行吗？
单头注意力把所有的表达能力都压在一组Q、K、V上，它只能学到一种”相关性”的定义。多头的意思是，把同样的输入投影到不同的子空间，每个头在自己的子空间里学到一种不同的相关性一-有的头学语法关系，有的头学语义关系，有的头学位置关系，最后拼起来，表达能力比单头强得多。
多头注意力的总计算量和单头比，有没有增加?
没有。因为每个头的维度是总维度除以头数，八个头的话每个头维度是原来的八分之一，总计算量和单头一样，但表达能力更强。这是一个很漂亮的设计。

## Layer Normalization放在哪里，Pre-norm和Post-norm的区别？
Post-norm是原始Transformer 用的，LayerNorm在残差连接之后。训练初期梯度流相对不稳定，需要比较小心地设置学习率。
Pre-norm是把LayerNorm放在子层之前，梯度可以直接通过残差连接反传，训练更稳定，现在大部分大模型用的都是Pre-norm。
但有研究发现，Pre-norm在收敛之后的最终性能，有时候不如Post-norm-Pre-norm让训练更容易，但可能牺牲了一点最终效果的上限。所以现在也有一些工作在研究怎么结合两者的优点。
能把这个trade-off说清楚的人，一定真正做过实验，真正在两种选择之间纠结过。

## RoPE+位置编码是什么，为什么比原来的正弦余弦编码好，好在哪里？
RoPE(Rotary Position Embedding) 是把位置信息通过旋转矩阵的方式编码进Q和K里，而不是直接加到token embedding上。
好处在于：它天然地让注意力分数只依赖于两个位置之间的相对距离，而不是绝对位置。两个token的内积，经过RoPE之后，只和它们的位置差有关一-这个性质叫相对位置感知，是RoPE的核心优势。
另外RoPE可以通过插值的方式外推到比训练时更长的序列，这个特性在长文本任务里很重要，也是Llama系列选择RoPE的重要原因之一。
原始的正弦余弦编码虽然也能外推，但外推性能下降得比RoPE快。

## Loss在训练初期下降很快，然后突然飙升，又慢慢降下来，这是什么情况，你会怎么处理?
这叫loss spike，大模型训练里很常见。原因可能是某个batch里有质量特别差的样本，梯度被带飞了。也可能是学习率过高，或者在某个特定的学习率调度节点上步长太大。处理方式：
第一步看是不是数据问题，把触发 spike 的那个 batch 找出来看看，通常能找到几条很异常的样本。
第二步看 gradient norm，如果某个 step 的 gradient norm异常大，说明梯度本身出了问题，可以适当调低梯度裁剪的阈值。
第三步看学习率调度，spike 有没有规律地在某些 step 出现，如果有，可能是调度设计的问题。
能说到这个细节的人，一定盯着训练曲线发过很多次呆。

## 指令微调之后模型变笨了，原来能做的任务现在做不好了，你怎么看这个问题?
这是灾难性遗忘。微调让模型在新任务上表现变好，但冲淡了预训练学到的通用能力。解决方向有几个：
一是在微调数据里混入一部分通用能力的数据，不要只有指令微调数据，防止模型遗忘太多。
二是用更小的学习率，更保守地更新参数，减少对预训练权重的破坏。
三是LoRA+这类参数高效微调方法--只更新少量适配器参数，原始权重冻住，天然地减少了遗忘，但有时候效果也有损失。
四是看具体任务，有些任务的遗忘是真实的问题，有些任务表面上看是遗忘，其实是评估方式不对，换一种评估方式发现模型还是会，只是说法变了。

## 模型量化，INT8和FP16精度损失的来源是什么?
量化的精度损失，主要来自两个地方：
一是权重分布不均匀。量化是把一个浮点数映射到一个整数，如果权重分布很集中，量化误差小；如果分布很分散，有些权重很大、有些很小，量化的分辨率不够用，误差就大。
二是 outlier。大模型里某些层的激活值里会出现少数极大的值，这些 outlier 会影响量化的 scale 因子，导致正常范围内的值精度下降。
LLM.int8() 这篇论文发现了这个问题，提出了混合精度量化的方案,把包含 outlier 的维度用 FP16 处理，其他维度用 INT8，在精度和效率之间做了一个平衡。
outlier即离群值，指在数据集中与其他大部分数据显著不同的观测值，其数值明显偏离数据的整体分布范围。

## 偏好数据怎么构造，两个候选答案的好坏差距很小的时候，标注标准是什么?
这个问题问的是你有没有真的设计过标注规范，有没有遇到过边界case，有没有在”这两个答案哪个更好”这件事上真的纠结过。
没做过的人，会说一些很高层的原则一-”更准确的更好，更有用的更好”。这些原则都对，但在两个答案都差不多准确、都差不多有用的时候，它们回答不了你的问题。
做过的人，会告诉你他们具体的标注维度一-准确性、相关性、流畅性、安全性分别怎么打分，优先级是什么，有没有绝对否决项。会告诉你哪些case是真的很难标，最后怎么处理有争议的对，是扔掉还是保留还是降权。

## 一个模型在测试集上表现很好，上线之后用户投诉很多，你觉得可能是什么原因，你会怎么排查?
分布漂移是最常见的原因一-测试集和真实用户的数据分布不一样，测试集代表不了真实场景。
排查思路是先看线上的输入数据，和测试集比较，找出分布差异。然后采样一批线上的badcase，人工看，找规律。然后看是不是某一类输入系统性地表现差，而这类输入在测试集里出现得很少。
另一个常见原因是评估指标没有对齐用户感受--你在优化一个和用户实际满意度不太相关的指标，指标好看，用户不买账。
第三个原因是标注标准和用户期望不一致--你的goldlabel是标注者认为的好答案，但用户想要的东西和标注者不一样。

## Lora 相关问题？
● 问：为什么 BB 矩阵要全零初始化？不能 AA 和 BB 都用高斯分布初始化吗？
  ○ 答：（重点！）如果都用随机初始化，在训练的第0步，LoRA 分支就会产生一个随机的噪声输出。这个噪声加到原来预训练好的大模型输出上，会导致模型第一步的 Loss 剧烈震荡（所谓的 Catastrophic Forgetting）。BB 全零初始化保证了 B×A=0B×A=0，也就是微调开始时，模型等价于原模型，保证了平滑的起步。
● 问：为什么是 BB 全零、AA Kaiming？反过来行不行？
  ○ 答：反过来（AA全零，BB随机）在数学上也是 B×A=0B×A=0，也是可以的。但官方选择了前者。
● 问：公式里的缩放因子 scale = alpha / r 到底有什么用？
  ○ 答：这是为了解耦超参数 rr 和学习率。我们在实验中如果想尝试不同的秩（比如从 r=8r=8 换成 r=32r=32），AA 和 BB 相乘后的矩阵数值量级会随着 rr 变大而变大。除以 rr 可以保证在使用不同 rank 时，LoRA 分支输出的方差基本保持一致，这样我们就不需要每次换 rank 都重新调一次学习率（Learning Rate）。
